# v0.2.0 Plan: Smart Warnings & Cache Awareness

## Key Insight

Claude Code's statusline already gives us cache tokens, cost data, and rate limits on every assistant message. We don't need to build custom token counting -- we need to make better use of what we're already collecting.

Native Claude Code commands (/cost, /context) are snapshot-only with no trends, no per-tool attribution, no cache TTL awareness. That's where CAT adds value.

## Feature Map

### Wave 1 (parallel, no dependencies)

| # | Feature | Files | Effort |
|---|---------|-------|--------|
| 1 | Context warnings suggest /compact and /clear | context_warnings.py | Small |
| 2 | Large tool output alerting (>5K tokens) | post_tool_use.py, _hook_config.py, config.py | Small |
| 3 | Overhead ratio in statusline ("cost: 3.2x fresh") | statusline.py | Small |
| 4 | Cache miss detection (5-min TTL expired) | delta_engine.py (SessionState) | Medium |

### Wave 2 (parallel, after Wave 1)

| # | Feature | Files | Effort |
|---|---------|-------|--------|
| 5 | Cache efficiency % in dashboard | models.py, routes.py, tui.py | Medium |
| 6 | PreCompact/PostCompact hooks + compaction tracking | New hooks, schema v6, new DB module, routes, tui | Large |

### Wave 3 (after Wave 2)

| # | Feature | Files | Effort |
|---|---------|-------|--------|
| 7 | Burn rate projection ("fills in ~X turns") | New engine module, models.py, routes.py, tui.py | Medium |

## Architecture Notes

- Features 1-3 are hook-side only (no collector changes)
- Feature 4 adds state to SessionState dataclass (collector-side)
- Features 5+7 both modify SessionSummary model (must coordinate)
- Feature 6 is the largest: new hooks, schema migration v6, new DB module, route handling
- All features use existing infrastructure (pending_messages for warnings, statusline for display)
